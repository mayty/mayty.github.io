ЛАБОРАТОРНАЯ РАБОТА №2

 

Цель: получить практические навыки использования расширенных возможностей коммутаторов, работающих в связке.

Лабораторная работа выполняется студентами индивидуально (на базе эмулятора КС, по варианту) или коллективно (на базе реального оборудования, каждый студент настраивает свое оборудование в классе).

Задание на лабораторную работу может содержать следующие пункты:

1. Согласно варианту задания реализовать сетевую топологию.


Рисунок -- Пример топологии

Вершины считать коммутаторами. Топология содержит 8 коммутаторов и до 12 каналов. Соединить коммутаторы, задействовав соответствующее количество портов.

2. Среди коммутаторов выбрать один (Root) -- возможно с учетом следующих пунктов задания.

3. Реализовать концепцию виланов 802.1Q, при этом учесть номер варианта. К двум топологически наиболее удаленным коммутаторам подключить шесть (общее количество) пользовательских станций. Одну из станций включить в административный вилан для управления всеми коммутаторами. Оставшиеся станции распределить между двумя пользовательскими виланами, так чтобы к каждому из крайних коммутаторов были подключены станции из обоих виланов. Предусмотреть native-вилан с идентификатором, отличным от идентификатора по умолчанию. Запретить передачу по транкам пакетов из неизвестных виланов.

4. Коммутатор Root соединить с каждым из соседних коммутаторов двумя каналами. Согласно варианту задания настроить Link Aggregation применительно к каждой паре каналов. При этом параметры физического соединения (скорость и режим) задать вручную.

5. Убедиться в работоспособности Spanning Tree. В каждом из виланов коммутатор Root принудительно назначить корневым мостом.

6. Для адресации в виланах использовать соответствующее количество подсетей из указанных в варианте задания. К коммутатору Root подключить дополнительную станцию-маршрутизатор. Настроить Inter-VLAN Routing и тем самым обеспечить достижимость всех сетевых интерфейсов.
